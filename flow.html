Section 1
User Roles
Five roles control what each user can see and do. Admin always has full access regardless of permission toggles.
Admin
Full access. Manages config, roles, dividends, rebates, and all approvals.
Staff
Day-to-day operations. Approves deposits, loans, savings. Cannot change config or roles.
Board of Directors
Read-only financial oversight. Views reports, members, loans, branch portfolio.
Collector
Creates batch deposits on behalf of multiple members. Limited access.
Member
Self-service. Applies for loans, submits deposits, manages savings and transfers.

Section 4
Savings Account
Auto-created once a member completes their first share. Members can deposit and withdraw with admin approval.
💳 Savings Deposit
Min ₱500 per deposit. Weekly cap configurable in System Config.
  • Member submits deposit request (amount, method, receipt)
  • Admin/Staff reviews and approves
  • Balance credited to savings account immediately
  • Ledger entry recorded (savings_deposit)
🏧 Savings Withdrawal
Cannot exceed current balance. Admin verifies before releasing.
  • Member requests withdrawal with reason
  • Admin validates balance is sufficient
  • Admin approves → balance deducted
  • Ledger entry recorded (savings_withdrawal)
📈 Savings Interest Release
Default rate: 2.5% per 6 months. Configured in System Config → savings_interest_rate.

💸 Repayment Recording
Members can record their own payments.
  • Member goes to Loan → Loan Detail
  • Clicks "Record Payment" — enters amount, method, reference
  • Repayment posted to schedule (installments marked paid)
  • Outstanding balance decreases. Ledger entry created.
✓ Members can record their own repayments — no admin action needed.

Section 12
Known Flow Gaps
Areas where the current implementation requires workarounds or manual staff involvement.
⚠ Member Cannot Self-initiate Share
Admin must open the first share subscription in Member Detail page. No member-facing "Apply for Share" form exists.
Workaround: Member contacts admin → admin opens share → member submits deposit requests.
⚠ Damayan Has No Self-pay
Members see their assessment amount but have no button to pay it. Staff must manually record each payment.
Workaround: Member pays staff in person or via transfer → staff records payment in Admin → Damayan.
⚠ No Guided Onboarding
After profile completion, members land on Dashboard with no clear next step. The share-opening process is invisible to them.
Workaround: Communicate the steps verbally or via printed onboarding guide.
⚠ No Dividend/Rebate Double-release Guard
The system shows the last release date as a warning, but does not block running dividends or rebates twice for the same period.
Workaround: Admin must manually verify the last release date before proceeding.
